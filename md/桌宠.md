# 【超简单】给你的网页领养一只 Live2D 看板娘 (live2d-widget 保姆级教程)

觉得自己的网页太单调？只有冷冰冰的代码和文字？是时候给它注入灵魂了！本文将教你如何通过开源项目 live2d-widget，只需几行代码，就能把初音未来、2233娘、黑猫等二次元角色“请”到你的网页上，还能陪你聊天、换装、玩游戏！

------

## 什么是 live2d-widget？

简单来说，它是一个基于 Web 的 Live2D 挂件。以前我们要自己在网页上写 Live2D 需要处理复杂的 Canvas 和模型加载逻辑，而 `live2d-widget`（特别是 stevenjoezhang 大佬维护的版本）把这些都封装好了：

- **自带灵魂**：有对话气泡，会根据你复制文字、点击链接做出反应。
- **自带交互**：支持换装、拍照、小游戏。
- **开箱即用**：一行 HTML 代码就能跑。

------

## 极速上手：30秒召唤看板娘

如果你不想折腾，只想最快速度看到效果，直接在你的 HTML 文件 `<body>` 标签结束前，加上这一行代码：

HTML

```
<!DOCTYPE html>
<html>
<head>
    <title>我的主页</title>
</head>
<body>
    <h1>欢迎来到我的二次元空间</h1>

    <script src="https://fastly.jsdelivr.net/gh/stevenjoezhang/live2d-widget@latest/autoload.js"></script>
</body>
</html>
```

保存并刷新网页（建议使用本地服务器 Live Server 打开），你会发现右下角出现了一只小白猫或者其他角色！

## 进阶配置：

默认的脚本有时会随机加载角色

要想**锁定角色**或**修改大小**，我们需要“劫持”它的配置。

### 1. 强制锁定角色

这个插件使用了浏览器缓存 (`localStorage`) 来记录你喜欢的角色。你可以在加载脚本前，强制写入配置：

HTML

```
<script>
    // 强制设定模型 ID (1 通常是 Miku，不同的后端 API ID 可能不同)
    // 建议先在界面上点刷新按钮找到 Miku，插件会自动记住
    try {
        localStorage.setItem('modelId', '1'); 
        localStorage.setItem('modelTexturesId', '0'); // 默认衣服
    } catch (e) {}
</script>
<script src="https://fastly.jsdelivr.net/gh/stevenjoezhang/live2d-widget@latest/autoload.js"></script>
```

### 2. 自定义提示语 (修改 waifu-tips.json)

觉得默认的台词不够个性？你可以把项目 fork 到本地，修改 `waifu-tips.json` 文件。

- **selector**: CSS 选择器（比如鼠标移到 `#GitHub` 按钮上）。
- **text**: 看板娘要说的话（比如“要去看看我的代码仓库吗？”）。

## 避坑指南 (血泪经验)

在使用过程中，你可能会遇到以下“灵异事件”，这里有解决方案：

### Q1: 为什么加载出来是乱码或者不显示？

- **原因**：你可能是直接双击 `.html` 文件打开的。
- **解法**：浏览器有安全策略（CORS），禁止本地文件跨域请求。请使用 VS Code 的 **Live Server** 插件，或者用 Node/Python 开启一个本地服务器。

### Q2: 手机上挡住内容了怎么办？

- `live2d-widget` 默认在手机端会自动变小或隐藏。如果你觉得还是挡路，可以在 CSS 里强制隐藏：

  CSS

  ```
  @media (max-width: 768px) {
      #waifu { display: none !important; }
  }
  ```

## 其他选择：Oh-My-Live2D

如果你是 Vue/React 开发者，或者觉得 live2d-widget 的风格太复古，可以试试更现代化的 oh-my-live2d。

它使用 TypeScript 重写，无需后端 API，配置更灵活：

JavaScript

```
// npm install oh-my-live2d
import { OML2D } from 'oh-my-live2d';
OML2D.loadOml2d({
  models: [{ path: '你的模型地址.json' }]
});
```
